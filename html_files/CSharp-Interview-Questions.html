Explain the difference between value types and reference types in C#

  • Value types store the actual value, and reference types store a reference to the value's memory location.
  • Value types include primitive types (e.g., int, float) and structures.
  • Value types are stored on the stack.
  • Reference types include classes, interfaces, arrays, and strings.
  • Reference types are stored on the heap, and the variable holds a reference to the memory location.
  • Assignment of a value type creates a copy of the value, while assignment of a reference type copies the reference, not the underlying object.
  • What is the purpose of the "using" statement in C#? How does it relate to IDisposable?

The "using" statement is used for automatic disposal of objects that implement the IDisposable interface.
IDisposable is used to release unmanaged resources like file handles, database connections, etc.
The "using" statement ensures that Dispose() method of IDisposable is called when the block of code is exited, even if an exception occurs.
It helps with resource management and prevents resource leaks.
What is the difference between an abstract class and an interface in C#?

An abstract class can have both defined and undefined (abstract) members, while an interface can only have undefined members (method signatures, properties, events).
A class can inherit from only one abstract class but can implement multiple interfaces.
An abstract class can have fields and provide default implementations, while an interface cannot have fields or implementations.
An abstract class is typically used when creating a base class with common functionality, while an interface is used for defining contracts that classes can implement.
How does exception handling work in C#? Explain the try-catch-finally block.

Exception handling allows you to catch and handle runtime errors gracefully.
The "try" block encloses the code that may throw an exception.
The "catch" block specifies the type of exception to catch and the code to execute when that exception occurs.
Multiple catch blocks can be used to handle different exceptions.
The "finally" block contains code that is executed regardless of whether an exception occurs.
It is used for cleaning up resources and is optional.
When an exception is thrown, the runtime searches for the first suitable catch block to handle it and executes the corresponding code.
What is the difference between "==" and "Equals()" in C#?

"==" is an equality operator used for comparing values.
"Equals()" is a method used for comparing the equality of objects.
For value types, "==" compares the values directly, while "Equals()" performs the value comparison.
For reference types, "==" compares the references, while "Equals()" performs the reference or value comparison depending on how it's implemented.
